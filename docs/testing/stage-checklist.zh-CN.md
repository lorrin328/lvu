# LVU 阶段性测试清单（MVP 闭环）

## 适用范围

- 分支：`codex/lvu-mvp-closure` 及后续合入版本
- 目标：验证本轮 5 个核心闭环能力是否可用

## 环境准备

1. 打开 Xcode，选择 iOS Simulator（建议 `iPhone 17`）。
2. 清理 App 历史数据（删除模拟器上的 App 后重装）。
3. 启动 App，确认示例数据已自动播种（Trips / Expenses 有初始内容）。

## 自动化前置检查（建议先跑）

```bash
tmp=/tmp/lvu-stage-check
rm -rf "$tmp" && mkdir -p "$tmp"
git archive HEAD | tar -x -C "$tmp"
cd "$tmp"
/tmp/xcodegen generate
xcodebuild -project LVU.xcodeproj -scheme LVU -destination 'platform=iOS Simulator,name=iPhone 17' test -quiet
```

通过标准：命令退出码为 `0`。

## 手工验收步骤

### 1. 消费关联 Trip / Day

- 操作：
1. 进入 `消费` Tab，点击 `添加凭证` -> `手动新增`。
2. 填写商户、金额、类别、地点、日期。
3. 在 `关联信息` 中选择某个 `所属旅行`，并选择一个 `关联行程日`。
4. 保存。
- 预期：
1. 新消费出现在列表中。
2. 该消费行显示对应旅行名与 `第 N 天`。
3. 返回该旅行详情页可看到该笔消费。

### 2. Trip 详情消费汇总（总额 / 按天）

- 操作：
1. 打开刚才关联的旅行详情页。
2. 滚动到 `消费` 分区。
- 预期：
1. 显示 `总消费`。
2. 显示按天小计（`第 1 天` / `第 2 天` 等）。
3. 若有未关联行程日的消费，显示 `未关联行程日` 小计。

### 3. 消费列表按 Trip / Day 过滤

- 操作：
1. 在 `消费` Tab 顶部筛选区，选择某个 `所属旅行`。
2. 再选择某个 `关联行程日`。
3. 切回 `全部旅行` / `全部行程日`。
- 预期：
1. 列表条目会随筛选条件变化。
2. 不匹配的消费不显示。
3. 切回全部后，记录恢复完整显示。

### 4. 路线编辑与删除

- 操作：
1. 进入某个旅行详情页的 `路线` 分区。
2. 对任一路线左滑，点击 `编辑路线`，修改距离/时长/备注并保存。
3. 再左滑点击 `删除路线`，确认删除。
- 预期：
1. 编辑后内容立即生效并持久化。
2. 删除前有确认弹窗。
3. 删除后该路线从列表消失，不会在重进页面后复现。

### 5. OCR 占位流程（上传占位 -> 解析草稿 -> 人工确认保存）

- 操作：
1. 进入 `消费` Tab，点击 `添加凭证` -> `OCR 占位`。
2. 选择一张图片（仅作占位），在 `凭证文本` 输入示例：
   `Little Panda Cafe` + `Total: 168.50` + 地址行。
3. 点击 `解析草稿`。
4. 检查草稿字段，必要时手工调整，选择 Trip/Day 后保存。
- 预期：
1. 页面显示已选图片状态。
2. 草稿字段会自动填充商户/金额（命中时）和置信度。
3. 保存后产生一条 `receiptDetected = true` 的消费记录。

## 通过判定

- 以上 5 项全部满足“预期”即判定本阶段通过。
- 如任一项失败，记录：
1. 失败步骤编号
2. 实际现象
3. 期望现象
4. 截图/录屏（可选）
